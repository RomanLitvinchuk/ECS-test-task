# ECS-test-task
Я использовал std::unique_ptr, так как в моём решении функционал shared_ptr в виде нескольких указателей на один и тот же компонент избыточен. Scene и MovementSystem могут получить доступ к указателям с помощью методов сцены, при этом никому другому эти указатели не нужны. Таким образом, unique_ptr хранятся в доступном месте для той логики, в которой они нужны, в единичном экземпляре.

В части 2 я выбрал сохранение и загрузку сцены. Я использовал json файлы и библиотеку nlohmann/json, так как, во-первых, для меня это удобнее, чем использовать обычный .txt. Во-вторых, данные об энтити и его компонентах естественным образом разбиваются на иерархию json файла.
